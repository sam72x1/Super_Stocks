## 🎯 الفكرة بجملة واحدة

- **🟢 جاهز للدخول الآن** = وضع الدخول المحسوب أصلًا (`entry_mode` بطبقة التفسير) يقول:
  «قرب الدعم» أو «بعد مسح مؤكَّد» — أي خطة الدفعات قابلة للتنفيذ الآن.
- **👀 متابعة** = «انتظار استعادة الدعم» أو «لا دخول (بعيد فوق المنطقة)» — مع ذكر السبب
  وما الذي يحوّله جاهزًا (رجوعه للمنطقة / استعادته الدعم).

**تحقّق حي (2026-07-08، القائمة النشطة 4 أسهم):** VFF وNAGE → 🟢 جاهز · LYEL وKSCP →
👀 متابعة (فوق منطقتها). **انقسام متوازن من أول يوم — لا قسم فاضٍ بنيويًا.** ولأن المرشّحين
الجدد يُرشَّحون أصلًا قرب دخولهم (رفض «بعيد_عن_الدخول» بالفرز)، أغلب الجدد يبدؤون 🟢.

⚠️ **درس تقاعد A/B (مثبَّت):** هذا الفصل **توقيت/قابلية تنفيذ** لا **جودة/تنبّؤ** — لا نزعم
أن «الجاهز» أكثر انفجارًا؛ نقول فقط: خطة الدخول قابلة للتنفيذ الآن أم لا. صفر أرقام جديدة.

## 📐 التصميم

### 1) المصدر الوحيد: دالة نقية `entry_status(s) -> dict`

توضع قرب `readiness_badge`. المدخل: نتيجة فرز `r` أو سجل قائمة `s`. المخرج:

```python
{"status": "ready_now" | "watch",
 "label":  "🟢 جاهز للدخول الآن" | "👀 متابعة",
 "reason": "<نص عربي مبسّط — فارغ للجاهز، سبب محدّد للمتابعة>"}
```

**المنطق (لا منطق جديد — قراءة `entry_mode` الموجود):**

```python
def entry_status(s):
    try:
        em = ((s.get("interp") or {}).get("entry_mode") or {})
        mode = em.get("mode")
        if mode is None:
            # احتياط السجلّات القديمة بلا interp: نفس صيغ build_interpretation حرفيًا
            lp = float(s.get("last_price") or s.get("price") or 0)
            trs = [float(x) for x in (s.get("tranches") or []) if x]
            _st = s.get("stop")
            stop0 = (float(_st[0]) if isinstance(_st, (list, tuple)) and _st
                     else (float(_st) if isinstance(_st, (int, float)) else None))
            piv = float(s.get("pivot") or 0)
            if lp <= 0 or not trs or piv <= 0:
                return {"status": "watch", "label": "👀 متابعة",
                        "reason": "بيانات ناقصة لتقييم وضع الدخول"}
            hi_z = max(trs)
            if stop0 and lp <= stop0:
                mode, reason = "no_entry_far", "كسر الوقف — الفكرة ملغاة/خطرة"
            elif lp < piv * 0.995:
                mode, reason = "reclaim_wait", "تحت الدعم — ننتظر استعادته"
            elif lp > hi_z * 1.05:
                mode, reason = "no_entry_far", "بعيد فوق منطقة الدفعات"
            else:
                mode, reason = "near_support", ""
        else:
            reason = em.get("reason") or ""
        if mode in ("near_support", "sweep_confirmed"):
            return {"status": "ready_now", "label": "🟢 جاهز للدخول الآن",
                    "reason": ""}
        if mode == "reclaim_wait":
            return {"status": "watch", "label": "👀 متابعة",
                    "reason": "تحت الدعم — يتحوّل جاهزًا باستعادته"}
        return {"status": "watch", "label": "👀 متابعة",
                "reason": (reason or "بعيد عن منطقة الدفعات")
                          + " — يتحوّل جاهزًا برجوعه لمنطقة الدفعات"
                          if "كسر الوقف" not in (reason or "") else reason}
    except Exception:
        return {"status": "watch", "label": "👀 متابعة",
                "reason": "تعذّر تقييم وضع الدخول"}
```

(لأوبيس: هذا **هيكل إرشادي** — حسّن الصياغة بما يطابق أسلوب الملف، لكن **التدفّق
والخرائط mode→status والنصوص الجوهرية كما هي**، والاحتياط ينسخ صيغ
`build_interpretation` حرفيًا: `pivot*0.995` · `max(trs)*1.05` · سقوط `last_price`.)

### 2) التقرير اليومي (`build_daily_message`)

- **الترويسة:** «N أسهم: X جاهز للدخول · Y متابعة (تجديد القائمة: الجمعة)».
- **قسمان** (partition وقت العرض — ترتيب `wl["stocks"]` المخزّن لا يُلمس، وداخل كل قسم
  يبقى الترتيب الحالي بالجاهزية):
  - `🟢 جاهز للدخول الآن (X)` — أسهم ready_now، كروتها **كما هي حرفيًا** (بلا حذف أي سطر).
  - `👀 متابعة (Y) — ينقصها التالي` — البقية، ويُضاف لكل سهم سطر واحد:
    `   👀 <السبب من entry_status>` (سطر واحد فقط، لا قائمة).
- الترقيم مستمر عبر القسمين (1..N). قسم فاضي → عنوانه لا يُطبع.
- سطر «🎯 في منطقة الدفعات الآن — نفّذ خطتك» الحالي **يبقى كما هو بشرطه الحالي**
  (إشارة التنفيذ الأدق داخل الجاهز).
- قسم الارتداد (W) **لا يُمسّ**.

### 3) كرت الترشيح (`build_message`) والفحص اليدوي

- سطر واحد بعد سطر «💪 القوة العامة»:
  - `🟢 جاهز للدخول الآن` أو `👀 متابعة — <السبب>`.
- `analyze_one` يرث السطر تلقائيًا عبر `build_message` (الحقول موجودة بمساره).

## ✅ اختبارات القبول (أضفها كلها)

1. **جاهز-منطقة:** interp.entry_mode=near_support → ready_now + label 🟢.
2. **جاهز-مسح:** mode=sweep_confirmed → ready_now (المسح المؤكَّد دخول عند فيصل).
3. **متابعة-استعادة:** mode=reclaim_wait → watch + السبب يحتوي «تحت الدعم».
4. **متابعة-بعيد:** mode=no_entry_far (سبب «بعيد فوق منطقة الدفعات») → watch +
   السبب يحتوي «يتحوّل جاهزًا برجوعه».
5. **متابعة-كسر وقف:** mode=no_entry_far بسبب «كسر الوقف…» → watch والسبب يذكر الوقف
   (بلا لاحقة «يتحوّل جاهزًا»).
6. **احتياط بلا interp:** سجل بلا interp لكن بـlast_price/tranches/stop/pivot:
   (أ) داخل المنطقة → ready_now · (ب) فوق `max(trs)*1.05` → watch — **بنفس صيغ
   build_interpretation** (قفل اتّساق: نفس المدخل يعطي نفس الحالة من المسارين).
7. **فاشل-آمن:** dict فاضي → watch «بيانات ناقصة…» (لا استثناء).
8. **اليومي:** قائمة فيها جاهز+متابعة → الترويسة «X جاهز للدخول · Y متابعة» + العنوانان +
   سطر 👀 لأسهم المتابعة فقط + الترقيم مستمر (السهم الأول بالقسم الثاني رقمه X+1).
9. **اليومي-قسم فاضٍ:** كلها متابعة → عنوان 🟢 لا يظهر · كلها جاهزة → عنوان 👀 لا يظهر.
10. **الكرت:** سطر الحالة يظهر (جاهز ومتابعة) والفحص اليدوي يطابق (عبر build_message).
11. **محاكاة القائمة الحية (تثبيت الاعتراض):** fixtures بأرقام VFF (1.95/دفعات حتى 1.91)
    → ready_now · LYEL (14.05/دفعات حتى 12.16) → watch — يثبت أن القسمين يمتلئان واقعًا.
12. **🔒 أقفال:** `entry_status` غير مذكورة في `rank_key`/`select_top`/`classify_tier`
    (getsource) · نصوصها بلا `≥≤><` · t1/t2/t3/الوقف لا تتغيّر · **الجاهزية غير مذكورة
    في entry_status** (قفل ضد رجوع عتبة 75 — getsource لا يحتوي `readiness` ولا
    `READY_PCT`).

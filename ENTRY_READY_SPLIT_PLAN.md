## 🎯 الفكرة بجملة واحدة

القائمة الحالية موحّدة («مؤهّل») مرتّبة بالجاهزية — المستخدم يبي **فصلًا عمليًا صريحًا**:
- **🟢 جاهز للدخول الآن** = الإعداد مكتمل **والسعر في منطقة التنفيذ** (لحظة فيصل العملية).
- **👀 متابعة** = سهم ارتكاز مؤهّل لكن **ينقصه شيء محدّد ليصير جاهزًا** — ويُقال له وش الناقص.

⚠️ **درس تقاعد A/B (لا تنساه):** هذا الفصل **توقيت/قابلية تنفيذ** (actionability) لا **جودة/تنبّؤ**
(quality). لا نزعم أن «الجاهز» أكثر انفجارًا — نقول فقط: خطة الدخول قابلة للتنفيذ **الآن** أم لا.
كل مكوّناته من إشارات موجودة محسوبة أصلًا (الجاهزية + منطقة الدفعات + الوقف/الدعم) —
**صفر أرقام سحرية جديدة، صفر ادّعاء تنبّؤي جديد.**

## 🔴 الخطوط الحمراء (أي خرق = رفض التنفيذ)

1. **طبقة عرض/تصنيف-عرضي فقط.** صفر مساس بـ: M1-M14 · `rank_key` · `select_top` (عضويتها
   مقفولة باختبار) · `classify_tier` · الدخول/الدفعات/الوقف/الأهداف/RR المخزّنة · فئة W.
2. **لا LOGIC_VERSION ولا force_renew.** الحالة تُحسب **وقت العرض** من حقول مخزّنة موجودة
   (دالة نقية) — لا تُخزَّن قيمة جديدة يعتمد عليها حساب. (يجوز تمرير الحالة في CSV كعمود
   تصديري إن سهُل — اختياري، ليس مطلوبًا.)
3. **لا عتبات جديدة.** المصادر الوحيدة:
   - عتبة الجاهزية: `CONFIG["READY_PCT"]` (=75، «🟢 جاهز» الموجودة).
   - منطقة التنفيذ: **نفس صيغة السطر الحالي حرفيًا** (سطر 4679):
     `min(trs) <= lp <= max(trs) * 1.005 and lp > stop`.
   - تحت الدعم: `lp < pivot * 0.995` (نفس عتبة `build_interpretation` الموجودة).
4. **قفل تطابق الفحص اليدوي:** `analyze_one` يعرض نفس الحالة بنفس الدالة.
5. **شكل الكرت المعتمد لا يُستبدل:** الإضافة = سطر حالة واحد لكل سهم + عنوانا قسمين
   بالتقرير اليومي. عربي مبسّط، **ممنوع ≥ ≤ > <** في أي نص معروض، RTL عبر `_rtl_join`
   (تلقائي لأن الأسطر تمر بنفس البنّاء).
6. **الحالة لا تدخل أي ترتيب اختيار.** التقسيم لقسمين في **التقرير اليومي فقط** (عرض)،
   وداخل كل قسم يبقى ترتيب الجاهزية الحالي. ترتيب `wl["stocks"]` المخزّن لا يُلمس.
7. **كل بند = اختبار** (قائمة القبول بالأسفل). قبل الدفع: `python3 test_bot.py` كامل
   (325+ كلها تنجح) + `pyflakes` نظيف + `ruff` بلا ملاحظات **جديدة** (خط الأساس 57).

## 📐 التصميم

### 1) المصدر الوحيد: دالة نقية `entry_status(s) -> dict`

توضع قرب `readiness_badge` (نفس عائلة العرض). المدخل: dict سهم (نتيجة فرز `r` أو سجل
قائمة `s` — الدالة تتعامل مع الاثنين). المخرج:

```python
{
  "status": "ready_now" | "watch",
  "label":  "🟢 جاهز للدخول الآن" | "👀 متابعة",
  "reasons": [ ... ]   # فارغة للجاهز؛ للمتابعة: أسباب محدّدة قابلة للفهم
}
```

**منطق القرار (بالترتيب، أول سبب يثبت يُجمَع — نجمع كل الأسباب لا أول واحد فقط):**

```python
lp   = s.get("last_price") or s.get("price")          # نفس سقوط build_interpretation
rdy  = s.get("readiness")
trs  = s.get("tranches") or []                        # لو فاضية: نفس احتياط اليومي
stop = s["stop"][0] لو tuple وإلا s["stop"]           # نفس معالجة build_interpretation
piv  = s.get("pivot")

reasons = []
لو lp غير متاح أو trs فاضية بعد الاحتياط:
    reasons.append("بيانات ناقصة (سعر/دفعات)")
وإلا:
    لو stop متاح و lp <= stop:
        reasons.append("تحت الوقف — الفكرة ملغاة (بانتظار الشطب)")
    لو piv متاح و lp < piv * 0.995:
        reasons.append("تحت الدعم — ننتظر استعادته")
    لو lp > max(trs) * 1.005:
        reasons.append("فوق منطقة الدفعات — ننتظر رجوعه لها")
    لو lp < min(trs) و lp > stop و ليس تحت الدعم:
        reasons.append("تحت منطقة الدفعات بقليل — قريب من الدخول")
        # (نادرة: بين الوقف وأدنى دفعة — معلومة صادقة لا تمنع «متابعة»)
    لو rdy is None:
        reasons.append("بيانات الجاهزية غير متاحة")
    وإلا لو rdy < CONFIG["READY_PCT"]:
        reasons.append(f"الجاهزية {rdy} من 100 (الجاهز يبدأ من {CONFIG['READY_PCT']})")

status = "ready_now" لو reasons فاضية وإلا "watch"
```

**ملاحظات إلزامية للتنفيذ:**
- «في المنطقة» = `min(trs) <= lp <= max(trs) * 1.005` — **انسخ الصيغة حرفيًا** من سطر
  4679 (شاملة هامش 0.005). الشرطان «فوق المنطقة/تحتها» هما نفيها بالاتجاهين.
- صياغة الأسباب **بلا علامات مقارنة** (خط أحمر #5) — كما في الأمثلة أعلاه.
- الدالة **فاشلة-آمنة**: أي استثناء → `{"status": "watch", "label": "👀 متابعة",
  "reasons": ["تعذّر تقييم الحالة"]}` (لا انهيار للتقرير أبدًا).
- **لا تقرأ** `interp` ولا الشبكة — حقول مخزّنة بسيطة فقط (تعمل للسجلّات القديمة).

### 2) العرض — التقرير اليومي (`build_daily_message`)

التغيير الوحيد بالبنية: **تقسيم حلقة الأسهم النشطة لقسمين** (بلا تغيير ترتيب داخلي —
نفس ترتيب `wl["stocks"]` الحالي، فقط partition وقت العرض):

```
📋 قائمة الأسبوع — 2026-07-09
7 أسهم: 2 جاهز للدخول · 5 متابعة (تجديد القائمة: الجمعة)

🟢 جاهز للدخول الآن (2)
1) 🎯 $AAA · 🟢 جاهز 82/100 · قوة 71
   ... (كرت السهم كما هو اليوم حرفيًا، بلا أي حذف) ...

👀 متابعة (5) — ينقصها التالي لتصير جاهزة
3) 🎯 $BBB · 🟡 يقترب 60/100 · قوة 55
   ...كرت السهم كما هو...
   👀 ينقصه: الجاهزية 60 من 100 (الجاهز يبدأ من 75) · فوق منطقة الدفعات — ننتظر رجوعه لها
```

- سطر «👀 ينقصه: ...» يُضاف **لأسهم المتابعة فقط** (الأسباب مفصولة بـ« · »، بحد أقصى
  3 أسباب الأولى — لا سطر طويل).
- سطر «🎯 في منطقة الدفعات الآن — نفّذ خطتك» الحالي **يبقى كما هو بشرطه الحالي**
  (معلومة منطقة مستقلة) — لا يُحذف ولا يُشترط بالجاهزية.
- الترقيم يستمر عبر القسمين (1..N) — لا يبدأ من جديد.
- لو أحد القسمين فاضي → لا يُطبع عنوانه (لا «جاهز (0)» فاضية).
- قسم الارتداد (W) الحالي **لا يُمسّ إطلاقًا** — هو أصلًا «متابعة ارتداد» مستقلة.
- ترويسة العدّ: «N أسهم: X جاهز للدخول · Y متابعة» (تحل محل «N سهم» الحالية فقط).

### 3) العرض — كرت الترشيح (`build_message`) والفحص اليدوي (`analyze_one`)

- سطر واحد يُضاف بعد سطر الرأس/القوة (قرب `readiness_tag`):
  - جاهز: `🟢 الحالة: جاهز للدخول الآن`
  - متابعة: `👀 الحالة: متابعة — ينقصه: <الأسباب بحد 3 مفصولة بـ« · »>`
- `analyze_one`: نفس السطر عبر **نفس الدالة** (يمرّ تلقائيًا لأنه يستخدم `build_message`
  — تأكّد فقط أن الحقول التي تقرأها الدالة موجودة بمسار الفحص اليدوي، وهي موجودة:
  price/tranches/stop/pivot/readiness).

### 4) الربط بالكود (نقاط الإدراج)

| المكان | التعديل |
|---|---|
| قرب `readiness_badge` (~4500) | تعريف `entry_status(s)` |
| `build_daily_message` (~4600) | partition + عنوانا قسمين + ترويسة العدّ + سطر «👀 ينقصه» |
| `build_message` (~3330) | سطر «الحالة» بعد سطر القوة العامة |
| `test_bot.py` | اختبارات القبول أدناه |
| `HANDOFF.md` + `CLAUDE.md` | تحديث الذاكرة بنفس الكومت (القاعدة الدائمة) |

**لا تعديل في:** scan_market · analyze_ticker · make_watch_entry · rank_key ·
select_top · classify_tier · update_watchlist_status · check_promotions · enrich ·
backtest_* · قسم الارتداد.

## ✅ اختبارات القبول (أضفها كلها)

1. **جاهز:** readiness=80 + lp داخل [min(trs), max(trs)*1.005] + فوق الوقف والدعم
   → `status="ready_now"` وreasons فاضية.
2. **متابعة-جاهزية:** readiness=60 داخل المنطقة → watch + سبب يحتوي «الجاهزية 60».
3. **متابعة-فوق المنطقة:** readiness=90 + lp فوق max(trs)*1.005 → watch + سبب
   «فوق منطقة الدفعات».
4. **متابعة-تحت الدعم:** lp تحت pivot*0.995 → watch + سبب «تحت الدعم».
5. **حدّية الهامش:** lp = max(trs)*1.005 بالضبط (وreadiness فوق العتبة) → **جاهز**
   (تطابق صيغة 4679 حرفيًا).
6. **فاشل-آمن:** dict فاضي/بلا سعر → watch + «بيانات ناقصة» (لا استثناء).
7. **اليومي:** قائمة فيها جاهز+متابعة → العنوانان يظهران + الترويسة «X جاهز للدخول ·
   Y متابعة» + سطر «👀 ينقصه» لأسهم المتابعة فقط + الترقيم مستمر.
8. **اليومي-قسم فاضي:** كل الأسهم متابعة → عنوان «🟢 جاهز» لا يظهر إطلاقًا.
9. **الكرت:** سطر «الحالة» يظهر بالكرت (جاهز ومتابعة).
10. **🔒 أقفال:** `entry_status` غير مذكورة في `rank_key`/`select_top`/`classify_tier`
    (getsource) · نصوصها بلا `≥≤><` · `set(select_top)` ثابتة (القفل الموجود يغطّي —
    لا تكسره) · t1/t2/t3/stop لا تتغيّر.
11. **سجل قديم:** سهم بلا tranches (يُبنى الاحتياط بنفس صيغة اليومي الحالية) → لا انهيار.

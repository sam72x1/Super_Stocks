## §2 — المشروع: ما هو ولماذا
بوت تيليجرام يفرز **«أسهم الارتكاز»** في ناسداك حسب منهجية المضارب فيصل (@kisar_).
**تعريف سهم الارتكاز:** انفجر تاريخيًّا ≥100% ثم انهار ≥50% (فينتج فلوت صغير + شورت منخفض + تقسيم
عكسي غالبًا) → الآن يتجمّع عند **قاع متشبّع** فوقه **فجوات سعرية** = هندسة مُهيّأة لانفجار جديد.
الهدف: التقاط هذه الأسهم **قبل** انفجارها وتنبيه المستخدم بنقاط الدخول/الوقف/الأهداف.

**المصدر الوحيد المعتمد** = كلام فيصل الموثّق بالصور والفيديو (`FAISAL_METHODOLOGY_NOTES.md` +
`FAISAL_IMAGES_CATALOG.md`). **لا اجتهاد شخصي** — المستخدم لا يتقبّل أي خطأ ولا فلسفة زائدة.

## §6 — طريقة التطوير (الانضباطات — التزمها في اقتراحاتك)
1. **المصدر = فيصل الموثّق فقط** — لا اجتهاد. أي قاعدة جديدة تحتاج سندًا من صوره/كلامه.
2. **مراجعة خصومية** (عدة عدسات: صحّة/أمان/تطبيع تقسيم/دلو مضاربي) قبل دمج أي تعديل ذي خطر.
3. **تسجيل مسبق (YAML)** لأي فرضية **قبل** رؤية نتيجتها — لا p-hacking (`PREREGISTERED_HYPOTHESES.yaml`).
4. **تجميد point-in-time** لأي باكتيست حكمي (لا تحكم على باكتيست حيّ — يقفز 26↔44%).
5. **بوابة الاختبار:** `test_bot.py` خروج 0 قبل أي دفع. الأقفال getsource تمنع الطبقات المساندة من دخول
   دوال الاختيار (`rank_key`/`select_top`/`classify_tier`/`entry_status`/`analyze_ticker`/`backtest_symbol`).
6. **مبدأ الطبقات:** عرض/بحث/باكتيست = آمن (بلا `LOGIC_VERSION`) · اختيار/دخول/وقف/أهداف = مقفول.
7. **قاعدة الذاكرة:** بعد كل قرار مهم، حدّث `HANDOFF.md` + `CLAUDE.md` بنفس الـcommit.
8. **انضباط الكوميت:** بند = commit · تحقّق بالعين قبل الدفع · تذييل محدّد · لا PR إلا بطلب المستخدم.
9. `LOGIC_VERSION` يُرفع **فقط** عند تعديل يمسّ الدخول/الوقف/الأهداف/المستويات (يشغّل ترحيلًا آليًّا).

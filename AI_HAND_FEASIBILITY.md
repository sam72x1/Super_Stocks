# 🤖🖐️ دراسة جدوى «اليد الاصطناعية» (وكيل AI ينوب عن يد فيصل)

> طلب المستخدم (2026-07-23، بعد أن أثبتنا أن الحافة = **طريقة فيصل** موجبة التوقّع): استكشاف
> وكيل AI متعلّم يحاكي يد فيصل (قرارات الدخول/الخروج/التعبئة). **الأمانة أولًا: هذه دراسة جدوى
> + برهان مبدئي، لا وعد بوكيل جاهز.** بحث فقط — صفر مسّ للإنتاج.

## ما الذي تفعله «اليد» ولا تفعله القاعدة الثابتة؟
القرار الأثمن الذي تتخذه يد فيصل ولا يقدر عليه وقفٌ ثابت: **هل هذا الهبوط تحت الدعم «مسح»
(كنس وقف مؤقت → أعِد الدخول) أم «كسر حقيقي» (اخرج)؟** موثّق كعلّة الوقف الثابت (§0-ح: «اليد
النشطة تمسح الوقف الثابت قبل الانطلاق»). هذا **تصنيف ثنائي**، لا RL كامل بالضرورة.

## الأفخاخ الأربعة (لماذا لا نبني وكيلًا كاملًا الآن)
1. **لا بيانات لحظية تاريخية للتدريب:** يد فيصل تقرأ الشريط (امتصاص عند قاع المسح). Polygon
   حيّ فقط؛ اللقطة يومية. وكيل على بيانات يومية أعمى عن جوهر القرار.
2. **أحداث نادرة:** انفجار 5-13% · ~74 انفجارًا/3 سنوات = أقل بكثير من حاجة سياسة RL
   (آلاف-ملايين حلقات) → overfitting/حفظ الناجين.
3. **انحياز البقاء:** التدريب على أسهم نجت → يحفظها، ينهار حيًّا.
4. **لا أمثلة خبير:** لا صفقات فيصل الفعلية للتقليد (imitation) → RL نقي على ضجيج = غير مستقر.

**الخلاصة:** وكيل RL كامل الآن = الفخّ الذي تجنّبناه (يلمع بالباكتيست، يخسر حيًّا). البناء
المسؤول **يبدأ بقياس هل هناك إشارة قابلة للتعلّم أصلًا**، ثم يجمع البيانات الصحيحة قبل أي نموذج.

## الطريق المنضبط (3 مراحل)
- **المرحلة 0 (الآن، مجانية):** برهان قابلية التعلّم على البيانات اليومية —
  `faisal_combo_backtest.py` وضع `FC_LEARN=1`. لكل **دخول-مسح** (من `sweep_entry_outcome`)
  نُخرِج ميزات point-in-time ونتيجة (ربح/خسارة)، ثم **تدريب 2023-2024 / اختبار 2025**
  (walk-forward، بلا تسريب): هل نسبة نجاح الربع الأعلى بالنموذج (OOS) تفوق الأساس؟
  **يضع السقف:** لو اليومي ≈ صفر OOS → قيمة الوكيل حصريًّا في التدفق اللحظي (يبرّر المرحلة 1).
  نموذج بسيط شفّاف (logistic بـnumpy، بلا sklearn) لمقاومة overfitting.
- **المرحلة 1 (لو 0 واعدة):** حصّاد بيانات حيّ — يسجّل كل مسح يراه الرادار + ميزات التدفق
  (`operator_flow`/`polygon_flow`: امتصاص، جدار الطلب، حصّة الدارك) + النتيجة → دفعة معنونة
  عبر أسابيع. نفس فلسفة E2-A (قياس prospective قبل النموذج).
- **المرحلة 2 (بعد بيانات كافية + موافقة):** الوكيل الفعلي — مصنّف/سياسة مُتحقَّقة على تدفق
  حقيقي، تُدمَج مع الرادار (طبقة التوقيت المُثبَتة) لا مع الاختيار. حواجز: OOS صارم، تسجيل مسبق.

## المبدأ الحاكم
لا نموذج إنتاجي قبل: (أ) إشارة قابلة للتعلّم مُثبَتة OOS · (ب) بيانات تدفق حيّة كافية (لا يومية
منحازة بقاءً) · (ج) تسجيل مسبق + موافقة المستخدم. **الوكيل يعزّز طبقة التوقيت (يد فيصل المُقنَّنة)
لا الاختيار** — لأن الحافة أُثبتت هناك.

## 📊 نتيجة المرحلة 0 (2026-07-23، لقطة `29935819997`) — ✅ ضوء أخضر

**النموذج المركّب (logistic نقيّة، تدريب 2023-24 / اختبار 2025):** نجاح الربع الأعلى بالنتيجة
**OOS = 14.7% مقابل أساس 8.5% → رفع 1.72× خارج العيّنة.** داخل التدريب 13.4% ≈ OOS 14.7% ⇒
**النموذج عمّم ولم يحفظ** (لا overfitting — الفجوة معكوسة، OOS ≥ التدريب). ✅ **حكم: إشارة قابلة
للتعلّم يوميًّا — يبرّر نموذجًا أغنى + المرحلة 1.**

**المعنى:** يوجد فعلًا محتوى قابل للتعلّم في قرار دخول-المسح (اختيار الدخولات الرابحة) —
**سقف أدنى** لأنه ميزات يومية-اختيارية بلا تدفق لحظي؛ جوهر يد فيصل (المسح مقابل الكسر =
الامتصاص) لم يدخل بعد، فالحافة الحقيقية أرجح أقوى. عند R:R ~14:1، رفع 1.72× في الاختيار =
قفزة توقّع كبيرة.

**الحدود:** سنة OOS واحدة (2025) · انحياز البقاء قائم (الرفع النسبي يخفّفه) · المغزى «الفكرة
تستحق البناء» لا «الوكيل جاهز».

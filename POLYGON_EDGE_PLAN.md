حصة دارك بول مرتفعة. هذي **فئة معلومات جديدة كليًا** لم يلمسها أي فحص — وهي مفهوم فيصل
«خلق السيولة الوهمية» مقاسًا من الصفقات نفسها بدل استنتاجه من MFI. **وPolygon يتيح تاريخ
الصفقات** → نقدر **نتحقّق قبل أن نثق** (تجربة مسجّلة مسبقًا بلا تسريب — الدرس الذي تعلمناه).

## 🔴 الخطوط الحمراء (أي خرق = رفض التنفيذ)

1. **الفاشل-الآمن المطلق (شرط المستخدم الحرفي):** كل دالة Polygon ترجع None عند: غياب
   `POLYGON_API_KEY` (يُقرأ **وقت النداء**) · 401/403 (اشتراك منتهٍ) · 429 (حد طلبات) · أي
   خطأ شبكي/JSON. والعرض عندها «—» أو غياب صامت للسطر. **صفر أثر على الفرز/التنبيهات
   الموجودة** — Yahoo يبقى المسار الأساسي والافتراضي لكل شيء قائم (Polygon **إضافة** فوقه).
2. **عرض/تسجيل فقط حتى يثبت الدليل.** درجة/مكوّنات التجميع **لا تدخل** rank_key/select_top/
   classify_tier/entry_status/analyze_ticker (قفل getsource لكل واحدة). أي وزن ترتيب =
   **فقط بعد** نجاح تجربة التحقق T-ACC بمعيارها المسجّل أدناه، وبموافقة المستخدم — درسا A/B
   والبصمة (§0-ز/§0-ح) ملزمان.
3. **لا LOGIC_VERSION** (كل هذا طبقة عرض/تشخيص/تحقق).
4. **لا درجة مركّبة مخترعة:** 3 مكوّنات خام تُعرض وتُخزَّن منفصلة (شراء عدواني % · طبعات
   كبيرة % · دارك %) — التجربة تحكم على **كل مكوّن على حدة** (لا تركيب أوزان قبل الدليل).
5. **العتبات:** أعِد استخدام الموجود (`BASE_WINDOW`=15 لنافذة القاع). عتبات العرض الجديدة
   الضرورية (طبعة كبيرة = 10× وسيط حجم طبعات السهم نفسه — **نسبية ذاتية المعايرة، لا رقم
   مطلق** · تنبيه بريماركت `PM_MOVE_PCT=10`) → CONFIG موثَّقة «عرض فقط، غير معايَرة».
6. **الاختبارات بلا إنترنت:** افصل المنطق النقي عن HTTP — دوال نقية تستقبل قوائم صفقات/شموع
   (dicts) وتُختبر بفكستشرات صناعية؛ أغلفة HTTP رقيقة فقط. كل مكوّن/مسار فشل = اختبار.
7. **ميزانية طلبات عاقلة:** حساب التجميع للمرشّحين المؤهّلين فقط (لا السوق كله)، بسقف
   30 رمزًا/تشغيل و`time.sleep(0.2)` بين الرموز + مهلة 12ث/طلب. التجربة التاريخية تطبع
   تقدّمها وتتحمّل الانقطاع (استئناف من CSV جزئي).
8. عربي مبسّط بلا ≥≤>< بالعرض · RTL عبر البنّائين الموجودين · تحديث `HANDOFF.md`+`CLAUDE.md`
   بنفس كل كومت · `python3 test_bot.py` كامل (416+ كلها تنجح) + pyflakes نظيف + ruff صفر
   جديدة قبل كل دفع. **كل جزء = PR منفصل** (أ ثم ب ثم ج) → دمج squash في main → مزامنة الفرع.

### أ-3) الربط والعرض (عرض فقط)
- `enrich`: للمرشّحين — `r["acc"] = silent_accumulation(sym)` (داخل try، بعد كتلة info؛

- **الكرت واليومي** (سطر واحد، يظهر فقط لو `acc` موجود):

### أ-4) 🔬 تجربة التحقق T-ACC (مسجّلة مسبقًا — قلب الجزء)
- **سكربت مستقل** `acc_verify.py` + workflow يدوي `acc_verify.yml`
  (inputs: `year` · env: `POLYGON_API_KEY` + أسرار تلغرام):
  1. يشغّل الباكتيست الموجود (`run_backtest` بوضعه القياسي) لسنة المدخل → قائمة إشارات
     (رمز + تاريخ إشارة + النتيجة/exploded — الحقول موجودة في trades).
  2. لكل إشارة: يجلب صفقات **نافذة BASE_WINDOW يومًا المنتهية يوم الإشارة** (بلا تسريب —
     الصفقات قبل الإشارة حصريًا) → `acc_components`. sleep 0.25 بين الرموز، طباعة تقدّم
     كل 10، وكتابة CSV تراكمي `acc_verify_<year>.csv` (استئناف لو انقطع: يقرأ الموجود
     ويتخطّى المحسوب).
  3. الحكم **لكل مكوّن على حدة** (شرائح ثلاثية كنمط `backtest_behav_correlation` حرفيًا:
     أدنى/أوسط/أعلى ثلث): معدل الانفجار + النجاح + فاصل Wilson لكل شريحة.
  4. **المعيار المسجَّل مسبقًا (لا يتغير بعد رؤية النتائج):** يُعتمد المكوّن فقط لو
     (فرق الانفجار بين الثلث الأعلى والأدنى ≥ 10 نقاط) **و**(فاصلا Wilson منفصلان)
     **و**(الاتجاه نفسه يصمد في السنتين 2025 و2026). وإلا: «يبقى عرضًا فقط».
  5. يرسل الحكم لتلغرام + يرفع CSV كـ artifact.
- **⚠️ حدود بيانات Polygon التاريخية:** لو تبيّن أن ticks التاريخية لا تغطي كامل 2025
  (حسب حدود الباقة) — **قلّص سنة التجربة لما هو متاح وقُل ذلك صراحة بالتقرير**، لا تفبرك.

### أ-5) اختبارات القبول (أ)
- `_tick_classify`: متتالية صاعدة → كلها +1 · هابطة → -1 · مساوية تحمل الاتجاه · الأولى 0.
- `acc_components`: فكستشر صفقات upticks كثيفة → aggressive_buy_pct مرتفع (فوق 60) ·
  فكستشر بطبعة 50× الوسيط → block_share يلتقطها · exchange=4 → dark_share ·
  أقل من 30 صفقة → None.
- بلا مفتاح → `silent_accumulation` None + الكرت/الفحص بلا السطر أو «—» (لا انهيار).
- 🔒 أقفال: `acc`/`silent_accumulation` خارج rank_key/select_top/classify_tier/
  entry_status/analyze_ticker (getsource) · عضوية select_top ثابتة (القفل القائم يغطي).
- `acc_verify`: دالة الحكم النقية (تستقبل قوائم صفقات جاهزة) تُختبر بفكستشرات:
  ارتباط مصنوع واضح → «يُعتمد» · مسطّح → «يبقى عرضًا» · نص المعيار المسجّل موجود بالتقرير.

  `*/30 11-21 * * 1-5` (التشغيلات المبكرة بلا مفتاح تنتهي فورًا بلا عمل — رخيصة).

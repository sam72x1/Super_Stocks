## النظام الحالي (مكتمل ويعمل)
1. **متابعة يومية ~10:23ص السعودية** (cron `23 7 * * 2-5`) + **تجديد أسبوعي الجمعة بعد الإغلاق**
   (cron `7 22 * * 5`، `RENEW_ON_CLOSE=1`) → قائمة **A/B**. ⏰ الدقائق غير المستديرة مقصودة
   (زحمة كرون GitHub بالدقيقة 0 كانت تؤخّر 2.2-3.6 ساعة/يوم — سجلّ 12 يومًا).
2. **14 بوابة** (M1–M14). الأساسية الصارمة (هوية الارتكاز) = **M1-M5** + أرضية RSI.
   - A = صفر نواقص تأكيد · B = ≤3 نواقص · أكثر = رفض.
3. **قائمة ثابتة دائمة**: تُضاف لها الجديد، **لا تُحذف إلا بستوب/هدف** (لا ترفرف).
   التجديد الكامل = **الجمعة 22:00 UTC بعد إغلاق السوق** (`should_renew(wl, force,
   renew_signal)` بإشارة `RENEW_ON_CLOSE` — شمعة أسبوعية مكتملة، طلب المستخدم
   2026-07-09). `run_daily_watchlist` = فرز+دمج (متابعة).
4. **قائمة مراقبة الارتداد** (مستقلة، `wl["pullback"]`): ارتكاز ارتفع فوق دخوله →
   متابعة يومية + تنبيه لحظي عند نزوله للدعم. مرتّبة بالجودة، حد 15.

5. **ترقية B→A** تلقائية + مؤشر صحة بيانات + retry تحميل (لا اختفاء عشوائي).
